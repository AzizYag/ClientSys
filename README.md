# ClientSys

**ClientSys** - настольная информационная система учета клиентов для
дипломного проекта «Разработка информационной системы учета клиентов».
Приложение предназначено для сотрудника организации: оно хранит клиентскую
базу, обращения, заказы, историю контактов и позволяет формировать
управленческую отчетность.

Приложение запускается отдельным окном на компьютере, работает с локальной
базой данных и не требует подключения к интернету или запуска сервера.

## Кратко о проекте

| Параметр | Значение |
| --- | --- |
| Название | ClientSys |
| Тип системы | Настольная информационная система |
| Назначение | Учет клиентов и работы с ними |
| Язык backend-логики | Go |
| Интерфейс | Wails + встроенный системный WebView |
| Хранилище данных | SQLite, локальный файл базы |
| Поддерживаемые ОС | Windows 10/11 x64, macOS Apple Silicon |
| Авторизация | Логин и пароль, пароли хранятся как bcrypt-хеши |
| Отчетность | Сводные показатели и экспорт CSV |
| Защита данных | Создание резервной копии SQLite-базы |

## Что делает приложение

ClientSys объединяет основной рабочий цикл обслуживания клиентов:

1. Сотрудник входит в систему под своей учетной записью.
2. В карточке клиента фиксируются ФИО, контакты, адрес, статус и примечания.
3. По клиенту создаются заявки с темой, описанием и этапом обработки.
4. Для оказанной услуги оформляется заказ со стоимостью и статусом выполнения.
5. Любые звонки, письма, встречи и консультации заносятся в журнал
   взаимодействий.
6. Руководитель или сотрудник выбирает период и получает показатели работы:
   количество клиентов, заявок, заказов, выполненных заказов, выручку и
   число контактов.
7. Отчет можно выгрузить в CSV, а базу данных - сохранить в резервный файл.

Такой сценарий позволяет отказаться от разрозненных записей и видеть всю
историю работы с клиентом в одной системе.

## Функциональные возможности

### Авторизация и сотрудники

- вход по логину и паролю;
- выход из учетной записи;
- самостоятельная смена текущего пароля;
- создание новых сотрудников администратором;
- назначение роли `manager` или `admin`;
- просмотр списка сотрудников, их логинов, ролей и даты создания.

Учетная запись администратора автоматически создается при первом запуске.
Создать другого сотрудника может только пользователь с ролью `admin`.
Операционные разделы системы после входа доступны авторизованным
сотрудникам.

### Рабочая панель «Обзор»

Начальный экран показывает актуальное состояние базы:

- общее количество клиентов;
- количество открытых заявок;
- число зарегистрированных заказов;
- выручку по выполненным заказам;
- последние клиенты;
- последние заявки;
- последние взаимодействия.

Экран удобен для презентации проекта и ежедневной быстрой оценки текущей
работы.

### Раздел «Клиенты»

В разделе ведется единый справочник клиентов. Для каждого клиента хранятся:

- фамилия, имя и отчество;
- телефон и электронная почта;
- адрес;
- дата регистрации;
- статус: `Новый`, `Активный`, `Неактивный`, `Черный список`;
- дополнительная информация и служебные примечания.

Доступные действия:

- добавление клиента;
- открытие и редактирование карточки;
- поиск по ФИО, телефону или электронной почте;
- фильтрация по статусу;
- удаление клиента.

При удалении клиента также удаляются связанные с ним заявки, заказы и
взаимодействия. Перед удалением приложение запрашивает подтверждение.

### Раздел «Заявки»

Заявка отражает обращение клиента и процесс его обработки. В ней указываются:

- клиент;
- тема обращения;
- описание проблемы или запроса;
- дата создания;
- статус: `Новая`, `В работе`, `Выполнена`, `Закрыта`;
- комментарий сотрудника.

Можно создавать, изменять, удалять заявки и фильтровать их по статусу.

### Раздел «Заказы»

Заказ фиксирует услугу, которая была предложена или оказана клиенту:

- клиент;
- наименование услуги;
- стоимость в рублях;
- дата оформления;
- статус: `Новый`, `В работе`, `Выполнен`, `Отменен`.

Сумма выручки в отчетах рассчитывается только по заказам со статусом
`Выполнен`.

### Раздел «Взаимодействия»

Журнал взаимодействий предназначен для сохранения истории коммуникаций с
клиентом. Запись включает:

- клиента;
- тип контакта: `Звонок`, `Письмо`, `Консультация`, `Встреча`,
  `Комментарий`;
- дату;
- результат или содержание контакта;
- сотрудника, который добавил запись.

Журнал помогает быстро восстановить контекст работы перед следующим
обращением клиента.

### Раздел «Отчеты»

Отчет формируется за выбранный период и содержит:

| Показатель | Как рассчитывается |
| --- | --- |
| Новые клиенты | Клиенты с датой регистрации в выбранном периоде |
| Заявки всего | Заявки, созданные в выбранном периоде |
| Открытые заявки | Заявки, не имеющие статуса `Закрыта` или `Выполнена` |
| Заказы | Заказы, оформленные в выбранном периоде |
| Выполненные заказы | Заказы со статусом `Выполнен` |
| Выручка | Сумма выполненных заказов |
| Взаимодействия | Контакты, зафиксированные в выбранном периоде |

Кнопка **«Экспортировать CSV»** сохраняет результат в файл
`clientsys_report.csv`. Файл создается в UTF-8 и открывается в табличных
редакторах, включая Microsoft Excel.

### Резервная копия

Кнопка **«Резервная копия»** в боковой панели сохраняет снимок базы данных в
файл `clientsys_backup.db`. Этот файл можно хранить отдельно и использовать
как копию всех введенных данных.

## Стартовые данные для демонстрации

При первом запуске с пустой базой ClientSys автоматически создает:

| Тип данных | Количество |
| --- | ---: |
| Учетная запись администратора | 1 |
| Клиенты | 8 |
| Заявки | 6 |
| Заказы | 6 |
| Взаимодействия | 8 |

Демонстрационные клиенты имеют разные статусы, а заявки и заказы находятся
на разных этапах обработки. Поэтому сразу после входа можно показать:

- заполненный экран обзора;
- поиск и фильтры в справочнике клиентов;
- редактирование карточек;
- работу с заявками, заказами и журналом контактов;
- отчет за период и экспорт CSV.

Если база полностью пустая, приложение автоматически добавит базовые
демонстрационные записи. Это сделано специально для защиты и проверки:
после удаления файла `clientsys.db` следующий запуск снова создаст
заполненную демонстрационную базу.

## Первый запуск

### Готовые файлы

Собранные версии расположены в каталоге `build/bin`:

```text
build/bin/ClientSys.exe                  - приложение для Windows x64
build/bin/ClientSys-Windows-x64.zip      - ZIP-архив Windows-версии
build/bin/ClientSys-macOS-arm64.zip      - ZIP-архив для macOS Apple Silicon
```

### Windows 10/11

1. Перенесите на Windows файл `ClientSys.exe` либо распакуйте
   `ClientSys-Windows-x64.zip`.
2. Запустите `ClientSys.exe`.
3. Если Windows запросит Microsoft WebView2 Runtime, установите его и
   повторите запуск.
4. Выполните вход под учетной записью администратора.

Windows-версия является portable-приложением: отдельная установка не
требуется. База `clientsys.db` создается автоматически рядом с файлом
`ClientSys.exe`.

### macOS Apple Silicon

1. Распакуйте `ClientSys-macOS-arm64.zip`.
2. Запустите `ClientSys.app`.
3. Выполните вход под учетной записью администратора.

Сборка предназначена для компьютеров Mac с процессором Apple Silicon
(`arm64`: M1, M2, M3 и более новые).

### Данные для входа

```text
Логин:  admin
Пароль: admin123
```

Сразу после первого входа рекомендуется нажать **«Сменить пароль»** и
установить собственный пароль длиной не менее 6 символов.

## Инструкция пользователя

### Добавление клиента

1. Войдите в систему и откройте раздел **«Клиенты»**.
2. Нажмите **«Добавить клиента»**.
3. Заполните обязательные поля: фамилия, имя и телефон.
4. При необходимости заполните e-mail, адрес и примечание.
5. Укажите статус клиента и нажмите кнопку сохранения.

### Создание заявки

1. Откройте раздел **«Заявки»**.
2. Нажмите кнопку создания заявки.
3. Выберите существующего клиента.
4. Укажите тему, дату, описание и статус.
5. Добавьте комментарий сотрудника и сохраните запись.

Заявку невозможно создать, пока в системе нет ни одного клиента.

### Оформление заказа

1. Откройте раздел **«Заказы»**.
2. Создайте новый заказ и выберите клиента.
3. Укажите услугу, сумму, дату и статус.
4. Для отражения дохода в отчете переведите завершенный заказ в статус
   **«Выполнен»**.

### Фиксация контакта с клиентом

1. Откройте раздел **«Взаимодействия»**.
2. Создайте новую запись.
3. Выберите клиента и тип контакта.
4. Укажите дату и результат коммуникации.
5. Сохраните запись; текущий сотрудник будет связан с ней автоматически.

### Формирование отчета

1. Откройте раздел **«Отчеты»**.
2. Выберите начальную и конечную дату периода.
3. Нажмите кнопку формирования отчета.
4. Просмотрите рассчитанные показатели.
5. При необходимости нажмите **«Экспортировать CSV»** и выберите место
   сохранения файла.

### Создание резервной копии

1. Нажмите **«Резервная копия»** в боковой панели.
2. Выберите папку и имя файла с расширением `.db`.
3. Храните полученный файл отдельно от рабочего компьютера, если данные
   используются в реальной эксплуатации.

## Хранение данных

Приложение не передает сведения во внешние сервисы. Все рабочие записи
сохраняются локально в SQLite-файле:

```text
macOS:   ~/Library/Application Support/ClientSys/clientsys.db
Windows: рядом с ClientSys.exe, файл clientsys.db
```

Для тестового запуска можно указать отдельный путь базы через переменную
окружения `CLIENTSYS_DB`:

```bash
CLIENTSYS_DB="$PWD/demo.db" ./build/bin/ClientSys.app/Contents/MacOS/ClientSys
```

Это удобно для демонстрации, поскольку не изменяет основную пользовательскую
базу.

## Структура базы данных

SQLite-база содержит следующие таблицы:

| Таблица | Назначение | Основные поля |
| --- | --- | --- |
| `users` | Учетные записи сотрудников | логин, e-mail, bcrypt-хеш пароля, ФИО, роль |
| `clients` | Клиентская база | ФИО, контакты, адрес, дата регистрации, статус |
| `requests` | Обращения клиентов | клиент, тема, описание, дата, статус, комментарий |
| `orders` | Заказы и услуги | клиент, услуга, дата, стоимость, статус |
| `interactions` | История контактов | клиент, тип, описание, дата, сотрудник |
| `app_metadata` | Служебные отметки приложения | признак однократного заполнения демо-данными |

Основные связи:

```text
users 1 ----- N clients
users 1 ----- N interactions
clients 1 -- N requests
clients 1 -- N orders
clients 1 -- N interactions
```

При удалении клиента связанные заявки, заказы и записи взаимодействий
удаляются каскадно. Для часто используемых выборок созданы индексы по
клиентам, датам и статусам.

## Безопасность и ограничения доступа

- Все рабочие разделы требуют предварительного входа.
- Пароли не хранятся в открытом виде: используется алгоритм `bcrypt`.
- Новый пароль должен иметь длину не менее 6 символов.
- Логин и электронная почта сотрудника должны быть уникальными.
- Только администратор может добавлять новые учетные записи.
- Резервная копия сохраняется только по выбранному пользователем пути.

Для учебной системы реализовано базовое разграничение: администратор
управляет сотрудниками, а вошедшие пользователи могут работать с
операционными данными.

## Технологическая реализация

Архитектура ClientSys состоит из трех частей:

| Уровень | Назначение | Реализация |
| --- | --- | --- |
| Представление | Окна, таблицы, формы, фильтры и модальные диалоги | Встроенный интерфейс Wails/WebView |
| Прикладная логика | Авторизация, операции разделов, экспорт и резервирование | Go |
| Данные | Локальное хранение и запросы | SQLite через Go-драйвер `modernc.org/sqlite` |

Приложение является desktop-программой: Wails создает собственное окно
приложения, в которое встроен пользовательский интерфейс. На Windows для
его отображения применяется системный WebView2, на macOS - системный WebView.
Бизнес-логика и доступ к базе выполняются в Go.

Основные исходные файлы:

```text
main.go                      - запуск окна приложения и подключение ресурсов
app.go                       - методы интерфейса, авторизация, отчет и backup
internal/model/model.go      - структуры данных
internal/store/store.go      - SQLite-схема, операции и демо-наполнение
internal/store/store_test.go - автоматические тесты слоя данных
frontend/dist/index.html     - разметка встроенного интерфейса
frontend/dist/app.css        - оформление интерфейса
frontend/dist/app.js         - поведение экранов и форм
wails.json                   - настройки сборки desktop-приложения
```

## Запуск из исходного кода

Необходимы:

- Go версии 1.25 или новее;
- Wails CLI v2.12.0;
- системные зависимости Wails для выбранной операционной системы.

Запуск в режиме разработки:

```bash
wails dev
```

Проверка слоя хранения:

```bash
go test -count=1 ./...
```

Сборка для текущей macOS-системы:

```bash
wails build -platform darwin/arm64 -o ClientSys
```

Сборка portable EXE для Windows x64:

```bash
wails build -platform windows/amd64 -o ClientSys.exe
```

Для формирования отдельного Windows-установщика через Wails требуется NSIS
и утилита `makensis`. Готовая версия проекта уже содержит переносимый
`ClientSys.exe`, который запускается без установки.

## Проверка проекта

В проекте предусмотрены автоматические тесты, которые подтверждают:

- создание базы и стартовых демонстрационных записей;
- авторизацию администратора и смену пароля;
- создание клиента, заявки, заказа и взаимодействия;
- расчет отчета и выручки;
- формирование резервной копии;
- каскадное удаление данных клиента;
- отсутствие повторного создания удаленных демонстрационных данных.

Готовые сборки дополнительно были проверены следующим образом:

- Windows-файл собран в формате `PE32+ executable (GUI) x86-64`;
- macOS-приложение подписано и проверено после распаковки архива;
- тестовый первый запуск macOS-сборки создал `8` клиентов, `6` заявок,
  `6` заказов и `8` взаимодействий.

## Формулировка для диплома

В пояснительной записке проект можно описать следующим образом:

> В ходе работы разработана настольная информационная система ClientSys,
> предназначенная для учета клиентов организации, регистрации заявок и
> заказов, ведения истории взаимодействий и формирования отчетности.
> Приложение реализовано на языке Go с использованием фреймворка Wails.
> Пользовательский интерфейс отображается во встроенном системном WebView,
> а обработка данных и бизнес-логика выполняются в Go. В качестве локального
> хранилища используется реляционная база данных SQLite. Система поддерживает
> авторизацию сотрудников, резервное копирование и экспорт отчетов в CSV.

## Сценарий демонстрации на защите

1. Запустить приложение и войти как `admin`.
2. Показать экран **«Обзор»** с уже заполненными показателями.
3. В разделе **«Клиенты»** выполнить поиск и открыть карточку клиента.
4. Создать нового клиента.
5. Для него добавить заявку и заказ.
6. Зафиксировать звонок или встречу в разделе **«Взаимодействия»**.
7. Открыть **«Отчеты»**, сформировать показатели за текущий период и
   выгрузить CSV.
8. Показать кнопку создания резервной копии базы.
9. В разделе **«Сотрудники»** продемонстрировать добавление учетной записи
   администратором.

Такой порядок демонстрирует полный цикл работы информационной системы:
регистрацию клиента, обработку обращения, учет услуги, историю коммуникаций
и итоговую отчетность.
